# PAPI Copy-n-Paste

## Project Context
This is a development platform for Polkadot API (PAPI). It generates TypeScript code for interacting with Polkadot/Substrate blockchains. The web app lets developers explore chain pallets, build queries and transactions, and copy working code snippets.

## Critical: PAPI, Not Legacy
- Use `polkadot-api` (PAPI v1.14+), NEVER `@polkadot/api` (legacy)
- Chain connections: Smoldot light client for relay chains, WebSocket for parachains
- Always use typed API: `client.getTypedApi(descriptor)`
- Descriptors are generated by running `npx papi` after adding chains

## Chains Supported
polkadot, kusama, westend, paseo, moonbeam, astar, acala, hydration, bifrost, rococo, paseo_asset_hub, polkadot_asset_hub, polkadot_people

## Provider Strategy
- **Smoldot** (relay chains): polkadot, kusama, westend, paseo — decentralized, no RPC dependency
- **WebSocket** (parachains): moonbeam, astar, acala, hydration, bifrost, rococo, paseo_asset_hub, polkadot_asset_hub, polkadot_people

## Hallucination Traps

AI models frequently generate wrong PAPI code. Every pattern below is WRONG — use the right column.

| Wrong (Legacy / Hallucinated) | Right (PAPI v1.14+) | Why |
|-------------------------------|---------------------|-----|
| `@polkadot/api` | `polkadot-api` | Legacy, deprecated |
| `ApiPromise.create()` | `createClient()` | Different client API |
| `api.query.balances.totalIssuance()` | `typedApi.query.Balances.TotalIssuance.getValue()` | PascalCase, explicit getValue |
| `api.tx.balances.transfer()` | `typedApi.tx.Balances.transfer_keep_alive({...})` | Plain `transfer` removed |
| `new Keyring()` | `sr25519CreateDerive` from `@polkadot-labs/hdkd` | No Keyring in PAPI |
| `new BN('1000000000000')` | `1_000_000_000_000n` (BigInt) | Native BigInt, not BN.js |
| `.signAndSend(pair)` | `.signAndSubmit(signer)` | Different method + signer |
| `import { WsProvider }` | `import { getWsProvider }` from subpath | Different provider API |
| Polkadot has 12 decimals | Polkadot has **10** decimals | Kusama has 12, not Polkadot |
| `chains/kusama` | `chains/ksmcc3` | Kusama spec is `ksmcc3` |

## Available Skills
- `/papi-connect` — Generate chain connection boilerplate (Smoldot or WebSocket)
- `/papi-query` — Generate storage query code (getValue, getEntries, watchValue)
- `/papi-tx` — Generate transaction code with wallet signing
- `/papi-subscribe` — Generate real-time subscription code with RxJS
- `/papi-scaffold` — Scaffold a new PAPI project from scratch
- `/papi-explore` — Explore chain pallets, calls, storage, events
- `/papi-staking` — Staking operations (bond, nominate, pools, rewards)
- `/papi-governance` — OpenGov voting, proposals, delegation
- `/papi-assets` — Asset Hub fungibles, NFTs, teleports
- `/papi-contract` — ink! and EVM smart contract interaction

## Code Patterns
```typescript
// Connection (Smoldot)
import { createClient } from "polkadot-api"
import { start } from "polkadot-api/smoldot"
import { getSmProvider } from "polkadot-api/sm-provider"
import { chainSpec } from "polkadot-api/chains/polkadot"
import { polkadot } from "@polkadot-api/descriptors"

const smoldot = start()
const chain = await smoldot.addChain({ chainSpec })
const client = createClient(getSmProvider(chain))
const typedApi = client.getTypedApi(polkadot)

// Query
const value = await typedApi.query.Balances.TotalIssuance.getValue()

// Transaction
const tx = typedApi.tx.Balances.transfer_keep_alive({
  dest: MultiAddress.Id("5Grw..."),
  value: 1_000_000_000_000n, // 1 DOT
})
const result = await tx.signAndSubmit(signer)

// Subscription
typedApi.query.System.Account.watchValue(accountId).subscribe(console.log)

// Cleanup
client.destroy()
smoldot.terminate()
```
